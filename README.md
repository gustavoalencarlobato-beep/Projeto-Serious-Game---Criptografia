---Projeto Serius Game Criptografia---
Os jogadores precisam aplicar manualmente essa tabela para converter das letras cifradas para o texto em português (sem acentos, em maiúsculas).
Essa escolha atende ao objetivo da disciplina de reforçar conceitos básicos de criptografia, sem entrar em algoritmos modernos como RSA.

Estrutura do Projeto
Os principais arquivos deste repositório são:	

•  index.html — Estrutura da interface do jogo (layout das abas, terminal, textos das fases).
•  style.css — Estilos visuais (tema escuro, painéis, tipografia estilo "terminal").
•  script.js — Lógica do jogo (controle de fases, verificação de respostas, mensagens de briefing e log).
•  Respostas corretas/Resposta correta.txt — Arquivo auxiliar com respostas ou anotações usadas no desenvolvimento.

Como Executar o Jogo

Não é necessário instalar nada especial.

1. Faça o clone ou download deste repositório.
2. Abra o arquivo index.html em um navegador moderno (Chrome, Edge, Firefox, etc.).
3. Use o painel Home para acompanhar o briefing e digitar as respostas.
4. Navegue pelas abas História: Segunda Guerra, Frente Ocidental, Batalha do Atlântico e Diário de Campo para encontrar as mensagens cifradas.


Alinhamento com os Critérios de Avaliação

1. Eficácia Educacional
•  Foco em cripto de substituição, um dos conceitos introdutórios mais importantes da disciplina.  
•  Mensagens estão sempre conectadas a contextos históricos reais (Segunda Guerra Mundial), reforçando a aplicação prática da criptografia.  
•  Cada fase funciona como um quebra-cabeça de decodificação, exigindo que o aluno aplique ativamente a tabela de substituição.

2. Jogabilidade e Engajamento
•  Interface em estilo terminal de agente secreto, com logs e briefings.
•  Dificuldade progressiva: começa com mensagens curtas e vai aumentando o tamanho e a complexidade do texto.
•  As narrativas criam um clima de missão militar, mantendo o jogador curioso sobre o que cada mensagem revela.

3. Qualidade da Apresentação
•  Código organizado em três arquivos principais (HTML, CSS, JS), facilitando a explicação em sala.
•  Narrativas e mensagens foram escritas pensando na coerência histórica e na clareza do objetivo educativo.
•  O repositório GitHub centraliza o código-fonte e pode ser apresentado junto com o GDD exigido pela disciplina.
